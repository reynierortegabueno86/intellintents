Respond with valid JSON only. No preamble, no markdown, no explanation.

## INTENTS

- state_risk_tolerance: User communicates their willingness to accept investment risk.
- state_investment_goal: User describes financial objectives (retirement, home, education, income).
- state_investment_horizon: User specifies the time frame for holding investments.
- state_financial_situation: User provides details about income, savings, debts, or net worth.
- state_experience_level: User describes prior investing knowledge and experience.
- state_esg_preference: User expresses interest or disinterest in ESG/sustainable criteria.
- update_risk_profile: User wants to change or refresh their previously established risk profile.
- refuse_to_answer_profiling: User declines to provide personal/financial information for profiling.

## DISAMBIGUATION

- "I'm conservative" -> state_risk_tolerance (describing risk appetite)
- "I want to update my risk level" -> update_risk_profile (changing existing profile)
- "I earn $100K/year" -> state_financial_situation
- "I want to retire in 20 years" -> could be state_investment_goal (retirement goal) or state_investment_horizon (20-year horizon). Prefer state_investment_goal when the PURPOSE is stated; prefer state_investment_horizon when only TIME is stated.

## UNKNOWN_SUBCATEGORY
